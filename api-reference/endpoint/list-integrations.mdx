## Overview

Returns all available integrations with key metadata you can use to render UI and decide how to connect. Typical use cases:

- Populate an integrations gallery or dropdown
- Show which services are supported and their auth types
- Drive onboarding flows before connecting and executing features

## Request

No parameters are required. This endpoint returns a full list.

<CodeGroup>

```bash
curl -X GET "https://api.incredible.one/v1/integrations"
```

```python
import requests

res = requests.get("https://api.incredible.one/v1/integrations")
print(res.json())
```

```javascript
const res = await fetch("https://api.incredible.one/v1/integrations");
console.log(await res.json());
```

</CodeGroup>

## Field reference

- id string — Unique identifier (e.g., `gmail`, `perplexity`).
- name string — Human‑readable name.
- description string — What the integration does.
- logo_url string — URL to the app logo to display in your UI.
- app_url string — Link to the app’s homepage.
- features array — List of available features/functions; each item has:
  - name string — Feature identifier (e.g., `GMAIL_SEND_EMAIL`).
  - description string — What the feature does.
- auth_method string — Authentication type: `OAUTH2`, `API_KEY`, `BASIC_AUTH`, or `NONE`.
