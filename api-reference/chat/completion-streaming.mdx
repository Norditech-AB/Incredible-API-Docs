Generate AI responses as a live stream using Server-Sent Events (SSE). This page focuses on streaming responses.

<Note>
  Set `stream: true` in your request body to enable streaming.
</Note>

## Quick Start

<CodeGroup>
```bash cURL
curl -X POST "https://api.incredible.one/v1/chat-completion" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-4",
    "messages": [
      { "role": "user", "content": "Tell me a story about AI" }
    ],
    "stream": true
  }'
```

```javascript JavaScript
const response = await fetch("https://api.incredible.one/v1/chat-completion", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    Authorization: "Bearer YOUR_API_KEY",
  },
  body: JSON.stringify({
    model: "gpt-4",
    messages: [{ role: "user", content: "Tell me a story about AI" }],
    stream: true,
  }),
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const chunk = decoder.decode(value);
  const lines = chunk.split("\n");

  for (const line of lines) {
    if (line.startsWith("data: ")) {
      const data = JSON.parse(line.slice(6));
      if (data.content === "[DONE]") {
        console.log("Stream completed");
        break;
      }
      console.log("Chunk:", data.content);
    }
  }
}
```
</CodeGroup>

## Request Parameters

<ParamField body="model" type="string" required>
  The model to use for chat completion.
</ParamField>

<ParamField body="messages" type="array" required>
  Array of message objects that make up the conversation.
</ParamField>

<ParamField body="stream" type="boolean" default="true">
  Enable streaming responses by setting this to `true`.
</ParamField>

<ParamField body="functions" type="array">
  Optional function definitions the AI can call.
</ParamField>

<ParamField body="integrations" type="array">
  Optional integrations to enable.
</ParamField>

## Response (Streaming)

When streaming is enabled, the response is returned as Server-Sent Events (SSE):

```
Content-Type: text/event-stream
Cache-Control: no-cache
Transfer-Encoding: chunked

data: {"content": "Hello"}

data: {"content": " there"}

data: {"content": "!"}

data: {"content": "[DONE]"}
```

## Error Responses

- `400`: Missing or invalid parameters
- `401`: Unauthorized (missing/invalid API key)
- `500`: Internal server error
