Run these minimal tests to verify your setup. For full runnable scripts, use our Cookbook (links below).

## Prerequisites

- An Incredible API key (if required by your deployment)
- curl installed

## Chat — non‑streaming smoke test

```bash
curl -X POST "https://api.incredible.one/v1/chat-completion" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "small-1",
    "stream": false,
    "messages": [{"role": "user", "content": "Hello Incredible."}]
  }'
```

Expected: JSON with `result.response[0].content`.

## Chat — streaming smoke test (SSE)

```bash
curl -X POST "https://api.incredible.one/v1/chat-completion/stream" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "small-1",
    "stream": true,
    "messages": [{"role": "user", "content": "Stream hello"}]
  }'
```

Expected: SSE lines, ending with `[DONE]`.

## Integrations — connect (API key)

```bash
curl -X POST "https://api.incredible.one/v1/integrations/perplexity/connect" \
  -H "Content-Type: application/json" \
  -d '{"user_id": "user_123", "api_key": "YOUR_API_KEY"}'
```

Expected: `{ "success": true, ... }`.

## Integrations — connect (OAuth)

```bash
curl -X POST "https://api.incredible.one/v1/integrations/gmail/connect" \
  -H "Content-Type: application/json" \
  -d '{"user_id": "user_123", "callback_url": "https://your.app/callback"}'
```

Expected: `{ "redirect_url": "https://..." }`.

## Integrations — execute a feature

```bash
curl -X POST "https://api.incredible.one/v1/integrations/gmail/execute" \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "user_123",
    "feature_name": "gmail_send_email",
    "inputs": {"recipient_email": "user@example.com", "body": "Hello"}
  }'
```

Expected: `{ "success": true, "result": { ... } }`.

## Troubleshooting

- Ensure `integration_id` exists before calling connect/execute.
- For OAuth, complete the redirect flow before executing features.
- Check model name in requests (`"small-1"`).
