The Incredible API has a unique authentication model. Here's what you need to know:

## üîë Getting an API Key

1. Visit [your dashboard](https://dashboard.incredible.one)
2. Navigate to API Keys section
3. Create a new API key
4. Copy the key (starts with `inc_`)

## üì° Using Your API Key

### Basic Usage (Currently Not Required)

The Incredible API currently works **without authentication** for basic usage. You can make requests without an `Authorization` header:

<CodeGroup>
  ```bash Shell
curl -X POST "https://api.incredible.one/v1/chat-completion" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "small-1",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
  ```

  ```python Python
import requests

response = requests.post(
    "https://api.incredible.one/v1/chat-completion",
    json={
        "model": "small-1",
        "messages": [{"role": "user", "content": "Hello!"}]
    }
)
  ```

  ```javascript JavaScript
const response = await fetch("https://api.incredible.one/v1/chat-completion", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    model: "small-1",
    messages: [{ role: "user", content: "Hello!" }]
  })
});
  ```
</CodeGroup>

## üîê When Authentication May Be Required

While basic usage works without auth, authentication may be required for:

- **Production applications**
- **Rate limiting control**
- **Usage tracking**
- **Advanced features**

## üìù Setting Up Authentication (If Needed)

### 1. Get an API Key

1. Visit [incredible.one](https://www.incredible.one)
2. Sign up for an account
3. Navigate to your dashboard
4. Copy your API key (starts with `inc_`)

### 2. Use the API Key

### With Authentication (Future Use)

When authentication is required, include your API key:

<CodeGroup>
  ```bash Shell
curl -X POST "https://api.incredible.one/v1/chat-completion" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer inc_your_api_key_here" \
  -d '{
    "model": "small-1",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
  ```

  ```python Python
import requests

response = requests.post(
    "https://api.incredible.one/v1/chat-completion",
    headers={"Authorization": "Bearer inc_your_api_key_here"},
    json={
        "model": "small-1",
        "messages": [{"role": "user", "content": "Hello!"}]
    }
)
  ```

  ```javascript JavaScript
const response = await fetch("https://api.incredible.one/v1/chat-completion", {
  method: "POST",
  headers: { 
    "Content-Type": "application/json",
    "Authorization": "Bearer inc_your_api_key_here"
  },
  body: JSON.stringify({
    model: "small-1",
    messages: [{ role: "user", content: "Hello!" }]
  })
});
  ```
</CodeGroup>

### 3. Environment Variables

```bash
# Linux/macOS
export INCREDIBLE_API_KEY="inc_your_api_key_here"

# Windows (Command Prompt)
set INCREDIBLE_API_KEY=inc_your_api_key_here

# Windows (PowerShell)
$env:INCREDIBLE_API_KEY="inc_your_api_key_here"
```

## üö® Error Handling

### Common Errors

**Model Not Found:**
```json
{
  "error": "Model not found"
}
```

**Invalid Request:**
```json
{
  "error": "Invalid request format"
}
```

## üí° Best Practices

1. **Start Simple**: Begin without authentication to test
2. **Add Auth Later**: Implement authentication when needed
3. **Secure Storage**: Use environment variables for API keys
4. **Monitor Usage**: Check your dashboard for usage patterns
