Authentication is the first step to using the Incredible API. This guide covers everything you need to know about API keys, security best practices, and handling authentication errors.

## üîë API Keys

### What is an API Key?

An API key is a unique identifier that authenticates your requests to the Incredible API. It's like a password that tells us who you are and what you're allowed to do.

### Getting Your API Key

1. **Sign Up**: Visit [incredible.one](https://www.incredible.one) and create an account
2. **Access Dashboard**: Log in to your account dashboard
3. **Generate Key**: Navigate to the API Keys section
4. **Copy Key**: Your API key will start with `inc_` (e.g., `inc_abc123def456...`)

## üîê Using Your API Key

### HTTP Header Authentication

The most common way to authenticate is using the `Authorization` header:

```bash
curl -X POST "https://api.incredible.one/v1/chat-completion" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer inc_your_api_key_here" \
  -d '{
    "model": "small-1",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

### Environment Variables (Recommended)

Store your API key as an environment variable:

```bash
# Linux/macOS
export INCREDIBLE_API_KEY="inc_your_api_key_here"

# Windows (Command Prompt)
set INCREDIBLE_API_KEY=inc_your_api_key_here

# Windows (PowerShell)
$env:INCREDIBLE_API_KEY="inc_your_api_key_here"
```

Then use it in your requests:

```bash
curl -X POST "https://api.incredible.one/v1/chat-completion" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $INCREDIBLE_API_KEY" \
  -d '{
    "model": "small-1",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

### Python Example

```python
import os
import requests

# Get API key from environment variable
api_key = os.getenv('INCREDIBLE_API_KEY')

if not api_key:
    raise ValueError("INCREDIBLE_API_KEY environment variable not set")

headers = {
    "Content-Type": "application/json",
    "Authorization": f"Bearer {api_key}"
}

response = requests.post(
    "https://api.incredible.one/v1/chat-completion",
    headers=headers,
    json={
        "model": "small-1",
        "messages": [{"role": "user", "content": "Hello!"}]
    }
)
```

## üö® Authentication Errors

### Common Error Responses

**401 Unauthorized - Invalid API Key:**
```json
{
  "error": {
    "message": "Invalid API key",
    "type": "authentication_error",
    "code": "invalid_api_key"
  }
}
```

**401 Unauthorized - Missing API Key:**
```json
{
  "error": {
    "message": "Missing API key",
    "type": "authentication_error",
    "code": "missing_api_key"
  }
}
```

**401 Unauthorized - Expired API Key:**
```json
{
  "error": {
    "message": "API key has expired",
    "type": "authentication_error",
    "code": "expired_api_key"
  }
}
```

**403 Forbidden - Insufficient Permissions:**
```json
{
  "error": {
    "message": "Insufficient permissions for this endpoint",
    "type": "authentication_error",
    "code": "insufficient_permissions"
  }
}
```

### Handling Authentication Errors

```python
import requests
from requests.exceptions import RequestException

def call_incredible_api(prompt):
    try:
        response = requests.post(
            "https://api.incredible.one/v1/chat-completion",
            headers={
                "Content-Type": "application/json",
                "Authorization": f"Bearer {os.getenv('INCREDIBLE_API_KEY')}"
            },
            json={
                "model": "small-1",
                "messages": [{"role": "user", "content": prompt}]
            }
        )
        
        if response.status_code == 401:
            error_data = response.json()
            if error_data.get('error', {}).get('code') == 'invalid_api_key':
                print("Invalid API key. Please check your credentials.")
            elif error_data.get('error', {}).get('code') == 'expired_api_key':
                print("API key has expired. Please generate a new one.")
            else:
                print("Authentication failed. Please check your API key.")
            return None
        elif response.status_code == 403:
            print("Insufficient permissions. Please check your account status.")
            return None
        elif response.status_code == 429:
            print("Rate limit exceeded. Please wait before making another request.")
            return None
        elif response.status_code != 200:
            print(f"API request failed with status {response.status_code}")
            return None
            
        return response.json()
        
    except RequestException as e:
        print(f"Network error: {e}")
        return None
    except ValueError as e:
        print(f"Invalid response: {e}")
        return None
```

## üîí Security Best Practices

### API Key Management

1. **Never Hardcode Keys**: Don't put API keys directly in your source code
2. **Use Environment Variables**: Store keys in environment variables or secure configuration files
3. **Rotate Keys Regularly**: Generate new keys periodically and revoke old ones
4. **Limit Key Scope**: Use different keys for different environments (development, staging, production)
5. **Monitor Usage**: Regularly check your dashboard for unusual activity

### Secure Storage Options

**Development Environment:**
```bash
# .env file (add to .gitignore)
INCREDIBLE_API_KEY=inc_your_api_key_here
```

**Production Environment:**
- **AWS**: AWS Secrets Manager or Parameter Store
- **Google Cloud**: Secret Manager
- **Azure**: Key Vault
- **Kubernetes**: Kubernetes Secrets
- **Docker**: Docker secrets or environment variables

### Example: AWS Secrets Manager

```python
import boto3
import json

def get_api_key_from_aws():
    client = boto3.client('secretsmanager')
    response = client.get_secret_value(SecretId='incredible-api-key')
    secret = json.loads(response['SecretString'])
    return secret['INCREDIBLE_API_KEY']

# Use the secret
api_key = get_api_key_from_aws()
```

### Example: Kubernetes Secrets

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: incredible-api-secret
type: Opaque
data:
  INCREDIBLE_API_KEY: <base64-encoded-api-key>
```

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  template:
    spec:
      containers:
      - name: my-app
        env:
        - name: INCREDIBLE_API_KEY
          valueFrom:
            secretKeyRef:
              name: incredible-api-secret
              key: INCREDIBLE_API_KEY
```

## üìä Rate Limiting and Quotas

### Understanding Limits

- **Rate Limits**: Maximum requests per minute/hour
- **Token Limits**: Maximum tokens per request
- **Data Limits**: Maximum file/data size per request
- **Function Call Limits**: Maximum function calls per request

### Checking Your Limits

```bash
# Check your current usage and limits
curl -H "Authorization: Bearer $INCREDIBLE_API_KEY" \
  "https://api.incredible.one/v1/usage"
```

### Handling Rate Limits

```python
import time
import requests

def call_api_with_retry(prompt, max_retries=3):
    for attempt in range(max_retries):
        try:
            response = requests.post(
                "https://api.incredible.one/v1/chat-completion",
                headers={
                    "Content-Type": "application/json",
                    "Authorization": f"Bearer {os.getenv('INCREDIBLE_API_KEY')}"
                },
                json={
                    "model": "small-1",
                    "messages": [{"role": "user", "content": prompt}]
                }
            )
            
            if response.status_code == 429:
                retry_after = int(response.headers.get('Retry-After', 60))
                print(f"Rate limited. Retrying in {retry_after} seconds...")
                time.sleep(retry_after)
                continue
                
            return response.json()
            
        except Exception as e:
            if attempt == max_retries - 1:
                raise e
            print(f"Attempt {attempt + 1} failed: {e}")
            time.sleep(2 ** attempt)  # Exponential backoff
    
    raise Exception("Max retries exceeded")
```

## üîç Troubleshooting

### Common Issues and Solutions

**"Invalid API key" Error:**
- Verify the API key is correct (starts with `inc_`)
- Check for extra spaces or characters
- Ensure the key hasn't been revoked

**"Missing API key" Error:**
- Add the `Authorization: Bearer <key>` header
- Check that the environment variable is set correctly
- Verify the variable name matches exactly

**"Expired API key" Error:**
- Generate a new API key in your dashboard
- Update your environment variables
- Check your account status

**"Insufficient permissions" Error:**
- Verify your account is active
- Check if you have access to the requested endpoint
- Contact support if the issue persists

### Debugging Steps

1. **Verify API Key Format**: Ensure it starts with `inc_` and is 64+ characters
2. **Check Environment Variables**: Use `echo $INCREDIBLE_API_KEY` to verify
3. **Test with cURL**: Try a simple request to isolate the issue
4. **Check Dashboard**: Verify the key is active in your account
5. **Review Logs**: Check your application logs for detailed error messages
